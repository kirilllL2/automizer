# Функциональные требования к приложению Automizer

## 1. Общие сведения

**Наименование системы:** Automizer - Система нормализации окон процессов и управления скриншотами

**Назначение:** Автоматизация управления окнами приложений (позиционирование и изменение размеров) и создание скриншотов с привязкой к пресетам конфигураций.

---

## 2. Функциональные требования

### 2.1. Управление окнами процессов

#### 2.1.1. Просмотр активных окон
| ID | FR-001 |
|---|---|
| **Наименование** | Отображение списка активных окон системы |
| **Описание** | Система должна отображать список всех активных окон операционной системы с информацией о заголовке, позиции и размерах |
| **Входные данные** | Запрос пользователя на обновление списка |
| **Выходные данные** | Таблица со столбцами: "Заголовок окна", "Позиция (X, Y)", "Размер (Ширина x Высота)" |
| **Приоритет** | Высокий |

#### 2.1.2. Поиск окон
| ID | FR-002 |
|---|---|
| **Наименование** | Фильтрация списка окон по названию |
| **Описание** | Пользователь должен иметь возможность фильтровать список окон по части заголовка окна |
| **Входные данные** | Текстовая строка поиска |
| **Выходные данные** | Отфильтрованный список окон, содержащих введенную подстроку в заголовке |
| **Приоритет** | Высокий |

#### 2.1.3. Выбор окна
| ID | FR-003 |
|---|---|
| **Наименование** | Выбор целевого окна для операций |
| **Описание** | Пользователь должен иметь возможность выбрать одно окно из списка для последующих операций |
| **Входные данные** | Клик по элементу списка окон |
| **Выходные данные** | Выбранное окно отображается как активное, его параметры загружаются в панель настроек |
| **Приоритет** | Высокий |

#### 2.1.4. Ручная настройка позиции и размеров
| ID | FR-004 |
|---|---|
| **Наименование** | Установка координат и размеров окна вручную |
| **Описание** | Пользователь должен иметь возможность задать точные значения координат (X, Y) и размеров (Ширина, Высота) для окна |
| **Входные данные** | Числовые значения в полях ввода: X, Y, Ширина, Высота |
| **Выходные данные** | Сохраненные параметры для применения к окну |
| **Приоритет** | Высокий |

#### 2.1.5. Предустановленные размеры (пресеты позиционирования)
| ID | FR-005 |
|---|---|
| **Наименование** | Быстрая установка стандартных расположений окна |
| **Описание** | Система должна предоставлять кнопки для быстрой установки распространенных конфигураций окна: половина экрана (слева/справа), четверть экрана (все 4 угла) |
| **Входные данные** | Нажатие соответствующей кнопки пресета |
| **Выходные данные** | Автоматическое заполнение полей X, Y, Ширина, Высота на основе текущего разрешения экрана |
| **Приоритет** | Средний |

#### 2.1.6. Нормализация окна
| ID | FR-006 |
|---|---|
| **Наименование** | Применение заданных параметров к окну |
| **Описание** | Система должна изменять позицию и размеры выбранного окна согласно указанным параметрам |
| **Входные данные** | Выбранное окно и заданные параметры позиции/размеров |
| **Выходные данные** | Окно перемещается и изменяет размеры согласно параметрам; сообщение об успехе/ошибке |
| **Приоритет** | Высокий |

## 3. Требования к интерфейсу

### 3.1. Структура приложения

| ID | UI-001 |
|---|---|
| **Наименование** | Многовкладочный интерфейс |
| **Описание** | Приложение должно иметь 4 вкладки: |
| **Состав** | 1. "Выбор процесса" - выбор окна и настройка позиции/размеров<br>2. "Пресеты процессов" - управление пресетами нормализации<br>3. "Скриншоты" - управление созданными скриншотами<br>4. "Пресеты скриншотов" - управление пресетами скриншотов и быстрый захват |
| **Приоритет** | Высокий |

### 3.2. Дизайн и стилизация

| ID | UI-002 |
|---|---|
| **Наименование** | Современный темный дизайн |
| **Описание** | Приложение должно использовать темную цветовую схему с акцентными цветами, современными шрифтами (Segoe UI), стилизованными элементами управления |
| **Приоритет** | Средний |

## 4. Нефункциональные требования

### 4.1. Производительность
| ID | NFR-001 |
|---|---|
| **Наименование** | Время отклика интерфейса |
| **Описание** | Операции поиска и фильтрации должны выполняться менее чем за 500 мс |

### 4.2. Надежность
| ID | NFR-002 |
|---|---|
| **Наименование** | Сохранение данных |
| **Описание** | Все пресеты и метаданные скриншотов должны сохраняться в постоянном хранилище и быть доступны после перезапуска приложения |

### 4.3. Удобство использования
| ID | NFR-003 |
|---|---|
| **Наименование** | Подтверждение деструктивных операций |
| **Описание** | Все операции удаления должны требовать подтверждения от пользователя через диалоговое окно |

## 5. Глоссарий

| Термин | Определение |
|---|---|
| **Нормализация окна** | Процесс изменения позиции и размеров окна до заданных параметров |
| **Пресет процесса** | Сохраненная конфигурация позиции и размеров для конкретного приложения/окна |
| **Пресет скриншота** | Сохраненная конфигурация области захвата скриншота с привязкой к пресету процесса |
| **Быстрый захват** | Комбинированная операция нормализации окна и создания скриншота по пресетам |
